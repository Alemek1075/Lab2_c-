ЗВІТ З ДОСЛІДЖЕННЯ (Варіант 12, inclusive_scan) Студент: Bazylevich Alex, К-27

    Бібліотечні політики: На масивах великого розміру (10М+ елементів) використання std::execution::par прискорює виконання у кілька разів порівняно зі звичайним seq, ефективно використовуючи багатоядерність.
    Власний паралельний алгоритм: Алгоритм ділить дані на K частин. При збільшенні K час виконання зменшується, доки K не перевищить кількість логічних ядер процесора.
    Оптимальне значення K: Найкраща швидкодія досягається, коли K дорівнює кількості апаратних потоків процесора. Якщо K значно більше (наприклад, у 10 разів), час зростає через накладні витрати на управління потоками.
    Висновок: Для великих даних паралельна обробка обов'язкова. Режим компіляції Release з оптимізацією є критичним для швидкодії.
